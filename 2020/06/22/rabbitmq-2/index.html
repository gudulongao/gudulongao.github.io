创建信道

注意要点

  • 信道非线程安全的，即信道Channel不可以在多线程之间共享。

Channel不能在线程之间共享，应用程序应该为每一个线程都开启一个新的Channel。Channel并发执行，会导致在网络出现错误的通信帧交错，同时也会影响发送方确认机制的运行。

  • 生产环境不要使用isOpen()方法。

当通过createChannel()创建信道后，信道就已经开启，无需通过isOpen来判断信道的状态，当我们在对信道做任何操作一旦信道被关闭，操作会直接抛SShutDownSignalException。同时在isOpen的源码中，使用到了同步锁，容易造成锁的竞争。

使用交换器和队列

API

需要额外关注的参数：

  • type:交换器类型。可以直接采用字符串(fanout,direct,topic和header),也可以采用枚举
  • autoDelete:是否自动删除，自动删除的前提是至少有一个队列或者交换器与这个交换器绑定，之后所有与这个交换器绑定的队列或者交换器都与此解绑。
  • internal:是否是内置。内置的交换器客户端不能直接发送消息到这个交换器。只能通过别的交换器转发到此交换器。
