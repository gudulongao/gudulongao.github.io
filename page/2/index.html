参数列表

参数 说明
accumulator 计算逻辑

从BiFunction函数接口种可以看到，其apply方法会根据注入的两个参数进行处理生成一个结果。入参和返回值的数据类型可以不一致。而BinaryOperator函数接口限制了数据类型，即BinaryOperator的apply入参和返回值都是一种数据类型。
所以方法Optional reduce(BinaryOperator accumulator) 的逻辑处理是 注入函数接口BinaryOperator实例，BinaryOperator的两个入参分别是上一次处理的结果和stream中下一个元素。这样我们只需要指定BinaryOperator的处理逻辑便可以根据stream中的所有元素来生成一个结果。

collect收集器

发表于 2019-08-19 | 分类于 java8 |

Stream的collect方法，即收集器，只要注入转换的方法，通过collect方法就可以基于stream中的元素来生成我们想要的数据结构。

辅助接口

Supplier

Supplier接口主要是提供方法用于获取指定类型的对象。

BiConsumer

  • accept(T,U) 提供对数据的处理逻辑
  • andThen(BiConsumer<T,U> after) 返回一个新的BiConsumer 其accept逻辑是先执行当前accept，成功后再执行after的accept逻辑。

样例

通过样例验证了 andThen是生成了一个新的BiConsumer实例，新的accept逻辑是先执行left的accept之后再执行after的accept。同时left的accept不能报错。

Function

  • R apply(T p) 根据输入参数 处理得到返回类型数据
  • compose(Function<K,T> before) 返回一个新的Function实例，新实例的apply先执行before 如果before执行无异常 before的输出结果作为原先apply逻辑的输入，最终执行结果作为返回结果。如果before执行报错则中断执行。
  • andThen(Function<R,V> after) 返回一个新的Function实例，新实例的apply先执行原有的apply逻辑，如果原有的apply无异常，则原先apply执行结果作为after的输入参数，after执行结果作为返回结果，如果原先的apply执行报错则中断执行。
  • identity(T t) 将输入结果作为输出结果。

样例


通过样例验证了compose方法和andThen的逻辑。

BiFunction

  • R apply(T t,U u) 根据参数t 和 u来处理获取返回类型数据
  • V andThen(Function<R,V> after) 执行apply逻辑，根据t 和 u生成结果R 再作为afterd的输入数据生成结果V 并返回。

BinaryOperator

  • minBy(Comparator c) 构建一个新的BinaryOperator实例，其apply逻辑是通过比较器获得两个输入参数中比较后小的值。

BinaryOperator是继承了函数接口BiFunction的，即根据两个输入参数来获取结果数据，由于要对输入的两个参数进行比较，所以参数类型要一致，故BiFunction的三个参数类型都是一样的

stream

发表于 2019-07-08 | 分类于 java8 |

JDK1.8 提供了流Stream的API。将数据元素集合看作流，流在管道中传输，并且可以在管道的节点对数据进行筛选、排序，聚合等处理。

什么是流

流(Stream)是来自数据源可以进行聚合操作的元素队列。

  • 数据源：流的来源，可以是集合，数组，I/O channel等。
  • 聚合操作：如filter、map、reduce、find、match、sort等。

流操作的特点

  • 针对流的操作都会立即返回流对象本身，这样多个流的操作会串联成一个管道，流式编码风格(fluent style)，便于对操作进行延迟执行和短路。
  • 集合之前的遍历都是通过迭代器iterator或者forEach的方式，显示的从集合外部进行迭代，而流采用访问者模式进行内部迭代。

内部迭代与外部迭代的差异

方式 可控 顺序 可并行
外部迭代 ✅ ✅ ❌
内部迭代 ❌ ❌ ✅

内部迭代的顺序不一定顺序，❌仅仅表示不一定。
内部迭代的可并行通过parallelStream()方法，可以并行操作，大幅度提升性能。

如何操作流

对于流的操作，其实主要就是由生成流、中间操作和最终操作三部分组成。
对于一次的流操作，可以有多个中间操作，有且只有一次最终操作。其中中间操作也称为转换操作，即多次中间操作是在对流数据进行转换，这种转换的效率并不受数据的量级影响，转换操作反而是一种惰性求值的过程。而有切只有一次的最终操作其实是将前面所有的转换处理一次执行，这一步会受到数据量级的影响，属于及早求值。

  • Collection接口和数组的默认方法，可以转换成Stream对象。

默认方法

默认方法：在接口中用default修饰的方法，接口中可以直接提供该方法的实现逻辑。即实现类可以不用实现该方法，直接使用接口中提供的实现方法。

默认方法的作用是解决接口的修改与现有的实现不兼容的场景。当接口发生变化时，其实现类很多，这个时候可以提供默认方法，实现类可以不用关注接口的变更。

子接口继承多个接口有相同的默认方法，或者实现类实现多个接口有相同的默认方法时，子接口和实现类必须实现相同的默认方法，否则会有编译错误。

  • 可以通过接口class.super.method来指定调用的默认方法。([className].super.[methodName])
  • 子接口或者实现类可以定义自己的默认方法对上层的默认方法进行覆盖。

不能对Object的方法提供默认方法即不能重写Object中的方法，可以重载。

接口静态方法。

  • 1.8版本接口可以声明静态方法。
  • 子接口或者实现类不会继承静态方法
  • static与default不能同时使用

Spring Data

Spring Data是Spring用于解决数据访问问题的一系列解决方案。包含大量关系型数据库及非关系型数据库的数据访问解决方案。
Spring Data 提供统一的API来进行数据访问操作而屏蔽了不同数据存储技术的差异。Spring提供Spring Data Commons项目无论使用关系型数据库还是非关系型数据库，都能够有Spring的统一标准，该标准提供CRUD(创建，获取，更新和删除)、查询、排序和分页的相关操作。
Spring Data Commons提供了Spring Data Repository抽象，使用Spring Data Repository可以减少数据访问层的代码。Spring Data Repository提供了基础的接口**Repository、CruRepository和PagingAndSortingRepository。

JPA(Java Persistens API)基于O/R映射的标准规范。即只定义标准规范（注解、接口），不提供实现，软件服务商根据标准规范来提供具体的实现，使用者只需要面向标准规范来使用不用与具体的软件服务商打交道。个人感觉通过JPA可以屏蔽不同的数据库的差异。

Spring Data JPA提供了JpaRepository接口。

SSL配置

SSL(Secure Sockets Layer)安全套接层。是为网络通信提供安全及数据完整性的一种安全协议。
SSL可分为两层：SSL记录洗衣和SSL握手协议。

协议 说明
SSL记录协议(SSL Record Protool) 建立在可靠的传输协议如TCP之上，为高层协议提供数据封装、压缩加密等基本功能的支持
SSL握手协议(SSL Handshake Protool) 建立在SSL记录协议之上，用于在实际数据传输开始前，通信双方进行身份认证、协商加密算法，交换加密密钥等

B/S的web应用中，HTTPS来实现SSL。HTTPS是以安全为目标的HTTP通道。即在HTTP下加入SSL层。

生成证书

使用SSL首先需要证书证书可以是自签名的，也可以是从SSL证书授权中心获取，演示通过自签名的方式来生成正式。

通过JDK自带的工具keytool在当前目录来生成.keystore 证书。
