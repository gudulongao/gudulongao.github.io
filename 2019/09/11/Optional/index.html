Optional

发表于 2019-09-11 | 分类于 java8 |

Optional是一个容器，可以很好的解决空指针问题。

JDK受Google Guava项目启发，引入了Optional类，Guava通过使用检查空值的方式来防止代码污染，它鼓励程序员写更干净的代码。Optional类已经成为Java 8类库的一部分。Optional实际上是个容器：它可以保存类型T的值，或者仅仅保存null。Optional提供很多有用的方法，这样我们就不用显式进行空值检测。

创建

从Optional源码中可以看到 构造函数是私有的，只能通过静态方法来创建Optional对象。其可以创建Optional实例的静态方法如下表所示：

方法名 说明
Optional empty() 获取空的Optional实例
Optional of(T value) 获取Optional实例，存储非空的value
Optional ofNullable(T value) 获取Optional实例。存储可以为空的value

API

empty

从Optional源码中可以看到 Optional中包含一个静态的EMPTY空内容的Optional实例。ofNullable方法一旦注入的参数是null则返回EMPTY。所以只要允许null的Optional都是EMPTY对象。故下面的测试所有的判断都是true。

map,flatMap

map和flatMap方法的源码如下图所示。

方法名 说明
Optional map(Function<T,U> mapper) 对Optional的值根据map的逻辑进行处理，处理结果是U 并将U存储到新的Optional容器中
Optional flatMap(Function<T,Optional> mapper) 对Optional的值根据map的逻辑进行处理，处理结果必须是Optional的数据类型

样例

从样例中可以看到，map方法的参数lambda表达式中返回的可以是任意类型的数据，因为map方法源码中最终返回的Optional结构是在map方法中自己包装的。而对于flatMap方法的参数lambda表达式中可以看到，必须返回一个Optional实例。flatMap源码中只是对mapper的执行结果进行了判空。

正确使用Optional


用Optional可以很优美的对参数进行判空，通过map方法来递进参数对象，当值为空通过orElse逻辑来获取默认值。
