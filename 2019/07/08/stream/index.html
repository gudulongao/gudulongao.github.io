stream

发表于 2019-07-08 | 分类于 java8 |

JDK1.8 提供了流Stream的API。将数据元素集合看作流，流在管道中传输，并且可以在管道的节点对数据进行筛选、排序，聚合等处理。

什么是流

流(Stream)是来自数据源可以进行聚合操作的元素队列。

  • 数据源：流的来源，可以是集合，数组，I/O channel等。
  • 聚合操作：如filter、map、reduce、find、match、sort等。

流操作的特点

  • 针对流的操作都会立即返回流对象本身，这样多个流的操作会串联成一个管道，流式编码风格(fluent style)，便于对操作进行延迟执行和短路。
  • 集合之前的遍历都是通过迭代器iterator或者forEach的方式，显示的从集合外部进行迭代，而流采用访问者模式进行内部迭代。

内部迭代与外部迭代的差异

方式 可控 顺序 可并行
外部迭代 ✅ ✅ ❌
内部迭代 ❌ ❌ ✅

内部迭代的顺序不一定顺序，❌仅仅表示不一定。
内部迭代的可并行通过parallelStream()方法，可以并行操作，大幅度提升性能。

如何操作流

对于流的操作，其实主要就是由生成流、中间操作和最终操作三部分组成。
对于一次的流操作，可以有多个中间操作，有且只有一次最终操作。其中中间操作也称为转换操作，即多次中间操作是在对流数据进行转换，这种转换的效率并不受数据的量级影响，转换操作反而是一种惰性求值的过程。而有切只有一次的最终操作其实是将前面所有的转换处理一次执行，这一步会受到数据量级的影响，属于及早求值。

生成流

  • Stream接口的静态工厂方法。
  • Collection接口和数组的默认方法，可以转换成Stream对象。
  • 其他

Stream静态工厂方法

方法名 说明
of(T… values) 返回有多个T元素的Stream
of(T value 返回有一个T元素的Stream
generator(Supplier s) 返回一个有无限深度的Stream,Supplier接口是JDK1.8新加的函数式接口，只有一个get方法用于返回T类型的实例。
iterate(final T seed, final UnaryOperator f) 返回一个无限深度的Stream，以T为种子，不断基于f的逻辑进行无限循环的方式构建Stream
empty() 返回一个没有元素的Stream

Collection接口和数组的默认方法

JDK1.8 提供了default 关键词，用于在已有的接口中拓展接口方法不用修改实现类。

Colllection

在Collection接口中提供了默认方法 stream和parallelStream方法可以生成流Stream。

Arrays

在Arrays中提供了静态的方法可以生成各种类型的Stream。

其他

Random

Random中提供了生成各种基本类型Stream的方法。

BitSet

BitSet中提供了stream方法来返回流。

Pattern

Pattern中提供了splitAsStream方法来返回流。

JarFile

JarFile中提供了stream方法来返回流。

中间操作

中间操作主要是针对Stream进行一些转换和限制操作。属于延迟计算。

concat

将两个流连接成为一个流。

  • 如果两个流有都是有序的，则新的流也是有序的。
  • 如果两个流中有一个是并行的，则新的流也是并行的。
  • 关闭新的流时，两个流也会关闭。

distinct

将Stream中重复的元素进行过滤.

filter

对Stream根据指定的条件进行过滤。

map

将Stream中指定的元素进行转换，返回的Stream中只保留转换后的元素。

Stream中已默认包含mapToDouble、mapToInt和mapToLong三个方法，即Stream默认支持将元素转换成Double、int和Long三种类型。

flatMap

flatMap与map类似，都是遍历Stream元素，并对元素做转换，不同之处是，并不是直接在Stream中用转换后的元素替换对应的元素。而是直接构造一个新的Stream。如果构造的新stream为空

peek

peek方法的参数是一个Consumer函数接口实例。其逻辑是在对Stream中的每一个元素在被消费的时候，都会事先执行Consumer逻辑。同时Consumer逻辑并不影响Stream中的元素。

样例中通过最终操作forEach来遍历Stream中的元素，而peek的执行逻辑会优先于forEach的操作。同时从输出结果可以看到，peek的Consumer逻辑处理结果并不影响原有的Stream元素。

skip

sikp方法的意义在于过滤Stream指定个数的元素。如果Stream的元素个数大于指定要过滤的个数。则返回一个length-n 个元素的新stream，否则返回一个空的stream。

sorted

对stream中的元素进行排序。

sorted方法有两个，一种是根据Object.equals(Object obj)方法来进行排序，另一个带有Comparaotr实例的是基于Compartor逻辑来进行排序。

最终操作

collect

count

统计stream元素格式

forEach

遍历stream中的元素。

forEachOrderd

如果stream是有序的，则有序的进行遍历如果是无序的则按照插入的顺序进行遍历。

max

找出stream元素排序后的最后一个元素。

min

找出stream元素排序后的第一个元素

reduce

allMatch

用于判断stream元素是否全部满足条件，如果全部满足返回true否则false。

anyMatch

用于判断stream元素中是否有一个元素满足条件，任意一个元素满足返回true否则false。

noeMatch

用于判断stream元素是否所有都不满足条件，如果都不满足返回true否则false。

findAny

用于获取含有stream元素的optional。如果stream为空，则optional为空，此操作不是确定的。并行操作时，可能会获取不同的结果，串行操作感觉是获取的第一个元素

findFirst

用于获取含有stream第一个元素的optional。如果stream为空则optional为空

limit

截取stream，如果limit保留的个数大于stream的元素个数，则返回stream全部元素，否则只保留指定个数的元素。
