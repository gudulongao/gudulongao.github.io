发表于 2019-06-30 | 分类于 springboot |

Spring Data

Spring Data是Spring用于解决数据访问问题的一系列解决方案。包含大量关系型数据库及非关系型数据库的数据访问解决方案。
Spring Data 提供统一的API来进行数据访问操作而屏蔽了不同数据存储技术的差异。Spring提供Spring Data Commons项目无论使用关系型数据库还是非关系型数据库，都能够有Spring的统一标准，该标准提供CRUD(创建，获取，更新和删除)、查询、排序和分页的相关操作。
Spring Data Commons提供了Spring Data Repository抽象，使用Spring Data Repository可以减少数据访问层的代码。Spring Data Repository提供了基础的接口**Repository、CruRepository和PagingAndSortingRepository。

Repository接口

Repository接口可以接受实体T 和主键id。

CrudRepository接口

CrudRepository接口提供CRUD操作。

PagingAndSortingRepository接口

PagingAndSortingRepository接口提供分页和排序操作。

Spring Data JPA

JPA(Java Persistens API)基于O/R映射的标准规范。即只定义标准规范（注解、接口），不提供实现，软件服务商根据标准规范来提供具体的实现，使用者只需要面向标准规范来使用不用与具体的软件服务商打交道。个人感觉通过JPA可以屏蔽不同的数据库的差异。

Spring Data JPA提供了JpaRepository接口。

Specification查询

Spring Data JPA支持基于准则查询。即Criteria查询。

  • Repository 需要继承接口JpaSpecificationExecutor
  • 自定义Specification，调用JpaSpecificationExecutor#findAll方法时注入自定义的Specification

分页排序

Spring Data JPA提供了Sort类、Page和Pageable接口来支持分页和排序。

在定义Repository的方法时，可以通过Sort来指定排序的属性。

Sort构造函数指定排序的顺序和需要排序的字段。

在定义Repository的方法时，可以通过Pageable 来指定分页信息。Pageable是接口，常用的实现是PageRequest。

SpringBoot的支持

JDBC配置

Spring Boot对JDBC做了自动配置。

  • spring.datasource 为前缀的属性自动配置datasource。
  • @EnableTransactionManagement注解开启事务支持。
  • 支持指定sql脚本来初始化表结构和数据。

JPA配置

  • SpringBoot默认的JPA实现是Hibernate。
  • 会自动扫描@Entity 的实体类。
  • 自动配置了OpenEntityManagerInViewInterceptor，并注册到SpringMVC的拦截器中。

SpringDataJPA的自动配置

  • SpringBoot默认自动开启了Spring Data JPA的支持，无需在配置类中通过@EnableJpaRepositories来开启支持。

ddl-auto配置项 说明
create 启动时，删除上一次生成的表，根据实体类来生成表结构。同时会清空表中的数据
create-drop 启动时根据实体类生成表，sessionFactory 关闭时表会自动删除
update 启动时根据实体类生成表，当实体属性变动时，会自动调整表结构，开发阶段常采用此选项
validate 启动时验证实体类与表结构是否一致，表结构稳定时采用此选项
none 不采取任何措施
